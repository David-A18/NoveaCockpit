Getting Started

Use this sample GUI to understand the main NoveaCockpit workflows before seeing the private desktop app.

Fake-data tour

Install the public demo

Download or clone the public repository.
Open sample-gui/index.html in a browser.
Optional: serve the folder locally for a browser-style URL.
cd sample-gui
python -m http.server 4174

Tour script

1. Go to Workbench. Click Preview plan, then Approve draft step, then Reset demo.
2. Open Runtime and Models to see the local-first boundary.
3. Open Dashboard to see fake operational metrics.
4. Open Interactions and Knowledge to understand the review workflow.

What this teaches

Agent Workbench: local plans, approvals, and audit-friendly run status. Try Preview plan → Approve draft step → Reset demo.
Runtime: desktop health, local provider readiness, and bounded process behavior.
Interactions: imported assistant sessions and safe review summaries.
Dashboard: fake operational metrics for runs, tokens, approvals, and error rate. In the private app these come from a local database.
Models: local model policy, approved model limits, and blocked oversized/cloud paths.
Knowledge: local vault context. Drafts stay database-only until a user explicitly applies them to the vault file.
